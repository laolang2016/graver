## 为什么选择 vscode

编辑器/IDE 有很多种, 比如 `Emacs` 、`vim/nvim` 、`clion` 、`codeblocks` 、 `visual studio` 、`qt creator` 等. 但是这些编辑器/IDE 都有这样那样的缺点, 比如收费, 比如某些常用的功能都需要写大量的配置才能勉强达到其他编辑器开箱即用的程度. 对我来说, `IDE` 一般用 `JetBrains` 或者宇宙第一 IDE `Visual Studio` 即可, 而编辑器则没有找到合适的, 因为我想要的编辑器要有如下特点:

1. 同时支持图形界面和字符界面, 且在这两种界面下的主要功能表现基本一致
2. 插件化架构
3. 配置文件使用 `lua` 而不是 `json` 或 `lisp`
4. 有类似 `.vscode` 的项目级别配置能力
5. 有类似 `vscode` 的 [便携模式](https://vscode.github.net.cn/docs/editor/portable) 的能力
6. 定制或写自己的插件要容易
7. 项目级别的插件启停配置
8. 项目级别的快捷键配置

出于易用性与配置速度考虑, 目前暂时使用 `vscode` 是最优解
